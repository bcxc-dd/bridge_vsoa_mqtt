# 智慧环境平台产品范围

## 目标与用户

平台用于三个项目的业务化联合展示和日常链路运维。普通用户查看环境和控制设备；测试运维员定位通信与性能问题；管理员维护账号、设备档案和连接配置。

核心业务流程为：`设备上报 -> MQTT Broker -> 协议桥接 -> VSOA -> 平台展示/告警`，下行为：`用户确认控制 -> MQTT 下发 -> 设备或桥接 ACK -> 平台记录结果`。

## 范围内

- LoRa、ZigBee 和协议桥接设备的统一登记、筛选和状态展示。
- 温度、湿度、烟雾、照明、电量和信号等可扩展业务数据。
- 单设备历史趋势、在线状态、告警、控制和 ACK 记录。
- 多 Broker、VSOA 连接、协议转换追踪和性能诊断。
- 本地账号、角色权限、深浅主题、审计和局域网访问。

## 明确不做

- 不为 16 个实验逐一建立测试页面、测试脚本或验收矩阵。
- 不代替三个小组编写 LoRa、ZigBee 或协议桥接核心业务代码。
- 不把平台生成的模拟转换结果当作真实项目代码输出。
- 不直接接收无线射频或串口数据；设备数据必须先进入约定 Broker 或服务。
- 第一版不实现房间、楼层、家庭成员和自动化场景编排。
- 不实现摄像头、图片中心和视频流处理。
- 不实现固件烧录、OTA 升级和设备生产配网。
- 不实现公网云部署、企业多租户、计费、SSO 或原生手机 App。
- 不用于安全关键或生产级工业控制。
- 没有消息序号时，不宣称能够准确计算网络层真实丢包率。
- 不要求所有设备具有相同数据格式，差异通过适配器和设备档案处理。

## 验收标准

- 三个项目的设备可在同一平台分类展示，多个 Broker 可同时接入。
- 用户可选择单设备查看实时数据、足够长度的历史趋势和最近通信状态。
- 至少一种 LoRa 传感设备与一种 ZigBee 控制设备完成真实上下行演示。
- MQTT 到 VSOA 的结果来自实际桥接项目，平台不伪造成功结果。
- 控制命令有确认、发送、ACK/失败状态和审计记录。
- 三种角色的菜单和 API 权限相互隔离。
- 深浅主题可用，刷新后账号会话、偏好、档案和历史数据不丢失。
- 局域网内其他电脑可使用本机 IP 访问平台。
